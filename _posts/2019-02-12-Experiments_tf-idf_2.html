Introduction

The first experiement in this series applied term frequency (tf) and term frequency - inverse document frequency (tf-idf) to a set of documents to discover hidden structure. We are using two documents on design thinking as our reference data. We found that tf-idf could help expose structure from a focused document like the Gasparanini paper. With the Stanford guide on design thinking, the focus of the paper was less clear from the term frequency output.

The big idea is to use output from TF-IDF analysis to discover what the documents are talking about, without reading the documents. This analysis will expose the document contents as bigrams, or pairs of 2 word sequences. Once the documents have been processed, the bigrams are exposed and a visual scan will reveal the indivudual document subject matter.

In general, an ngram is a grouping of terms from a source document. We are using bigrams, or ngrams of length 2 words. You can create ngrams of any length.

As with the first experiment we gather our data into a tidy format. The goal this time is to create a single tidy tibble of bigrams. This tidy tibble will contain all of the words for both documents. In this case, the tidy format will contain a pair of words for each row of the tibble.

Now that the data is in a tibble, let’s clean it and arrange it. Viewing the data at this point shows it to be arranged as 18 rows of strings of words. We can create bigrams from these strings using unnest_tokens. We then can clean up the tidy tibble to remove stop words and numbers.

The process is to create the rows of bigrams, separate the pairs of words in the bigrams, check and remove the stop words, numbers and then unite the result.

data("stop_words")

#
# create the ngrams
bow <- bag_of_words %>% 
  unnest_tokens(bigram, text, token = "ngrams", n = 2)

# break the bigrams apart so we can do further processing
bow_separated <- bow %>%
  separate(bigram, c("word1", "word2"), sep=" ")

#
# drop stop words and numbers
bow_filtered <- bow_separated %>%
  filter(!word1 %in% stop_words$word) %>%
  filter(!word2 %in% stop_words$word) %>%
  filter(!grepl(pattern="[0123456789]+", word1)) %>%
  filter(!grepl(pattern="[0123456789]+", word2))

# put the data back into a bigram formatted tibble
bow_united <- bow_filtered %>%
  unite(bigram, word1, word2, sep=" ")

#
# create a visual
bow_united %>%
  count(bigram, book, sort=TRUE) %>%
  bind_tf_idf(bigram, book, n) %>%
  arrange(desc(tf_idf)) %>%
  mutate(bigram = factor(bigram, levels = rev(unique(bigram)))) %>% 
  group_by(book) %>% 
  top_n(10) %>% 
  ungroup %>%
  ggplot(aes(bigram, tf_idf, fill = book)) +
  geom_col(show.legend = TRUE) +
  labs(x = NULL, y = "tf-idf") +
  facet_wrap(~book, ncol = 2, scales = "free") +
  coord_flip()
## Selecting by tf_idf

The result is an enhanced view of the content of each paper. On the left, the results show the Gasparanini paper to contain “Designerly Thinking”, “congitive empathy” and “user journey”. The paper on the right contains a more wide ranging set of frequent terms. This wider range better reflects the nature of the Stanford paper, since it is a “how-to” guide.

Conclusion

In this case, tf-idf combined with bigrams produced a good representation of the content of the papers in our set of data. Without having to read the papers we can see from the previous diagrams what the papers are targeted toward. After reading the papers, it is clear, the Gasparanini paper focused n developing empathy within the conference participants. The Stanford paper is not as focused. It is structured as a guide around design thinking in general.
